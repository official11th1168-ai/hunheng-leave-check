# ระบบเช็คสิทธิวันลาพนักงาน

เว็บแอปเชื่อมกับ Supabase project เดียวกับระบบใบลา (officialHH / Leave) มี 2 หน้า:

- `index.html` — หน้าเช็คสิทธิสำหรับพนักงาน (กรอกชื่อ-นามสกุล ไม่ต้องล็อกอิน)
- `admin.html` — หน้าจัดการข้อมูลพนักงาน + สรุป/รายงาน (ป้องกันด้วย PIN)

## ขั้นตอนที่ 1 — สร้างตาราง employees ใน Supabase

1. เข้า Supabase Dashboard > โปรเจกต์ officialHH > SQL Editor
2. เปิดไฟล์ `schema.sql` ในนี้ วางทั้งหมดแล้วกด Run
3. เช็คว่าตาราง `employees` ขึ้นมาใน Table Editor แล้ว

> ถ้าเคยรัน `schema.sql` ไปแล้วรอบก่อนหน้า (ก่อนมีหมวดช่างเย็บผ้า) ให้รัน `migration_tailor.sql` เพิ่มอีกไฟล์เดียว เพื่อเพิ่มคอลัมน์ `employee_type` และ `sewing_nickname` เข้าไปในตารางที่มีอยู่แล้ว (ไม่กระทบข้อมูลเดิม)

## หมวดพนักงาน 2 แบบ

- **รายเดือน** — มีสิทธิวันลา แสดงในแท็บ "พนักงานประจำ" ของหน้าเช็คข้อมูล
- **เหมารายชิ้น** — ช่างเย็บผ้า ไม่มีสิทธิวันลา แต่มีข้อมูลการส่งงานรายสัปดาห์ (จำนวนชิ้น, แยกเกรด A/B/C/BB, จำนวนรอบแก้งาน, รูปภาพ) ดึงสดจาก Supabase project **sewing-grade-system** (`rslsjllwrbgjdqavhmbk`)

เวลาเพิ่มช่างเย็บผ้าในหน้าแอดมิน ต้องเลือกประเภท "เหมารายชิ้น" แล้วกรอก **ชื่อเล่นในระบบเกรด** ให้ตรงกับที่ใช้ในระบบเกรดเป๊ะๆ (เช่น รส, พลอย, แมว, วิกกี้, กาน, วรรณ, น้อง) เพราะระบบจะ match ด้วยชื่อเล่นนี้ไปค้นตาราง `submissions.tailor_name` — ถ้าสะกดไม่ตรงจะหาข้อมูลไม่เจอ

**หมายเหตุความปลอดภัย:** หน้าเช็คสิทธิพนักงานไม่มีล็อกอินตามที่ขอ จึงเปิดให้ anon key อ่านตาราง `employees` และ `leave_requests` ได้แบบ public (RLS อนุญาต select ทั้งหมด) หน้าแอดมินป้องกันด้วย PIN แต่เป็นการเช็คฝั่งหน้าเว็บ (client-side) เท่านั้น ไม่ใช่การป้องกันระดับฐานข้อมูลจริง เหมือนกับระบบ SewingGrade/QC ที่เคยทำ — ถ้าใครรู้ URL ของ admin.html และเดา PIN ถูกก็แก้ข้อมูลได้ ถ้าต้องการความปลอดภัยที่สูงขึ้นในอนาคต แนะนำทำ Cloudflare Pages Function คั่นกลางเพื่อเช็ค PIN ฝั่งเซิร์ฟเวอร์ก่อนแก้ข้อมูลจริง (เหมือนที่ทำไว้ในระบบ stock)

## ขั้นตอนที่ 2 — ตั้งรหัส PIN ผู้ดูแล

เปิดไฟล์ `admin.js` บรรทัดแรก แก้ค่า:
```js
const ADMIN_PIN = "1168";
```
เปลี่ยนเป็นรหัสที่ต้องการก่อน deploy จริง (ค่าเริ่มต้นตอนนี้คือ 1168 เหมือนระบบ SewingGrade — แนะนำให้เปลี่ยนเป็นรหัสอื่นเพื่อไม่ให้ซ้ำกัน)

## ขั้นตอนที่ 3 — เพิ่มข้อมูลพนักงาน

เข้า `admin.html` > กรอก PIN > แท็บ "รายชื่อพนักงาน" > กด "+ เพิ่มพนักงาน"
กรอกให้ครบ โดยเฉพาะ **ชื่อ-นามสกุลต้องสะกดตรงกับที่พนักงานใช้กรอกตอนส่งใบลา** เพราะระบบ match กันด้วยชื่อ-นามสกุล (ไม่สนตัวพิมพ์เล็ก-ใหญ่ แต่ตัวสะกดต้องตรง)

## ขั้นตอนที่ 4 — Deploy ขึ้น Cloudflare Pages

1. สร้าง repo ใหม่บน GitHub เช่น `hunheng-leave-check`
2. push ไฟล์ทั้งหมดในโฟลเดอร์นี้ (ยกเว้น `schema.sql` กับ `README.md` ก็ push ไปด้วยได้ ไม่มีผลอะไร)
3. ไปที่ Cloudflare Pages > Create a project > Connect to Git > เลือก repo นี้
4. Build settings: ไม่ต้องตั้งค่า build command / output directory (เป็น static site ธรรมดา ปล่อย root ไว้)
5. Deploy แล้วจะได้ URL เช่น `hunheng-leave-check.pages.dev`

## กติกาสิทธิวันลาที่ตั้งไว้ในระบบ (แก้ไขได้ที่ `shared.js` ฟังก์ชัน `getEntitlement`)

| อายุงาน | ลากิจ | ลาป่วย | ลาพักร้อน | หมายเหตุ |
|---|---|---|---|---|
| น้อยกว่า 6 เดือน | 0 | 0 | 0 | ยังไม่ผ่านโปร ลาได้แต่ไม่ได้รับค่าจ้างวันลา |
| 6–11 เดือน | 1 วัน/ปี | 15 วัน/ปี | 0 | ไม่หักเงิน |
| ตั้งแต่ 1 ปีขึ้นไป | 2 วัน/ปี | 30 วัน/ปี | 15 วัน/ปี | สิทธิเต็ม |

"ปีสิทธิ" นับจากวันครบรอบวันเข้างานของแต่ละคน (ไม่ใช่ปีปฏิทิน) เช่น เข้างาน 15 มี.ค. 2567 → รอบสิทธิปีนี้คือ 15 มี.ค. 2569 – 14 มี.ค. 2570

ยอด "ใช้ไป" คำนวณจากใบลาในตาราง `leave_requests` ที่สถานะ = "อนุมัติ" และ `start_date` อยู่ในรอบสิทธิปัจจุบัน โดย match ด้วยชื่อ-นามสกุล
